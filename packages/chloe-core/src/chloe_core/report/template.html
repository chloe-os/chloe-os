Methodology

Overview of each pipeline stage

  1. Variant Calling & Filtering — Somatic variants are parsed from VCF files. Quality filters are applied (min quality {{ min_quality }}, min depth {{ min_read_depth }}, min AF {{ min_allele_frequency }}) and, when a matched normal sample is available, germline variants are subtracted.
  2. Variant Annotation — Each somatic variant is annotated with the affected gene, transcript, protein change, and surrounding peptide sequence using Ensembl VEP or equivalent tools.
  3. MHC Binding Prediction — Mutant and wildtype peptides of lengths {{ peptide_lengths }} amino acids are generated and evaluated for binding to {{ alleles_used | length }} DLA class I alleles using {{ predictor }}.
  4. Neoantigen Ranking — Candidates are scored using a composite metric incorporating binding affinity, agretopicity (mutant vs. wildtype binding ratio), variant allele frequency, and variant consequence severity. Top candidates are ranked by this score.
  5. Structural Analysis (optional) — Predicted 3D structure of peptide-MHC complexes to assess binding geometry and stability using protein language models.
  6. AI Interpretation (optional) — Large language model analysis of the results to provide context, literature references, and plain-language summaries.
  7. Report Generation — This self-contained HTML report is produced, combining all pipeline outputs into a format suitable for review by researchers and clinicians.
